## Markdown-Format

Jede Datei enthält:

```markdown
---
id: [WordPress ID]
title: "[Titel]"
slug: [URL-Slug]
type: post|page
status: publish
date: YYYY-MM-DD
modified: YYYY-MM-DD
link: [Vollständige URL]
categories: [IDs]        # nur bei Posts
tags: [IDs]              # nur bei Posts
parent: [Parent-ID]      # nur bei Pages mit Parent
excerpt: "[Auszug]"      # wenn vorhanden
---

# [Titel]

[Inhalt als Markdown]
```

### Konvertierung HTML zu Markdown

- Überschriften: `<h2>` -> `##`
- Listen: `<ul><li>` -> `- `
- Fett: `<strong>` -> `**...**`
- Kursiv: `<em>` -> `*...*`
- Links: `<a href="">` -> `[Text](URL)`
- Bilder: `<img>` -> `[Bild](URL)` (nur URL, keine Bilder gespeichert)
- Elementor-Wrapper wurden entfernt
- HTML-Entities dekodiert (ä, ö, ü, ß, €, etc.)

## Bekannte Einschränkungen

1. **Keine Bilder:** Per Entscheidung nicht gespeichert, nur URLs dokumentiert
2. **Elementor-Markup:** Teilweise noch in Pages enthalten (besonders bei komplexen Layouts)
3. **JavaScript:** Rechner-Seiten enthalten eingebettetes JavaScript im Markdown
4. **Yoast SEO:** Meta-Daten (SEO-Titel, Focus-Keyword) nicht über Standard-API verfügbar
